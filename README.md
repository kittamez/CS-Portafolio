CS 230 Project Design Document Q&A
Q: Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
A: Our client was The Gaming Room, a game development company looking to expand its reach across multiple platforms. Teh software was a cross-platform game application called Draw It or Lose It based on the show Win, Lose or Draw. the app needed to host multiple rounds, display 200 pictures over a minute clock, allow teams that ere behind a 15-second chance to steal answers, and connect players through devices and browsers.
Q: What did you do particularly well in developing this documentation?
A: I translated complex system architecture, software constraints, and networking protocols into clean, simple, and easily understood explanation that makes it easy to follow .
Q; What about the process of working through a design document did you find helpful when developing the code?
A: writing the design document forced me to map out constraints i would not have considered before, like the device RAM limitations and how to save as much memory as possible
Q: If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
A: Id add a architectural diagram graphic showing buyers how a Linux OS sets the threads to individual game sessions snd allows for better game management. I think a visual guide would be easier to understand than examples.

A: The user wanted a game that wasable to have a large library that able to be modified and adapted to devises, so instead of the game homding the images, they are turned to files that are downloaded from the server and will not cause the game to lag.
